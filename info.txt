# 🛒 E-Commerce Microservices Platform

A **production-style backend architecture** for an E-Commerce system built using **Java, Spring Boot, and Microservices architecture**.
This project demonstrates how large-scale applications structure services, manage communication, and handle infrastructure using modern backend technologies.

The goal of this project is to practice **real-world system design concepts** and build a portfolio-level backend project.

---

# 📌 Architecture Overview

The system is designed using **Microservices Architecture**, where each service is independent and communicates with others through APIs or event-driven messaging.

```
                Client
                  |
                  ▼
            API Gateway
                  |
      ---------------------------------
      |        |        |             |
      ▼        ▼        ▼             ▼
 User Service Product  Order       Payment
              Service  Service     Service
                                   |
                                   ▼
                            Notification Service
```

### Infrastructure Components

* **Service Discovery** → Eureka Server
* **API Gateway** → Central request routing
* **MySQL** → Main database
* **Redis** → Caching layer
* **Kafka + Zookeeper** → Event streaming between services
* **Docker Compose** → Infrastructure orchestration

# 3️⃣ Built Service Discovery (Eureka Server)

You created the **service-discovery** module.

This runs **Netflix Eureka**.

**What it does**

All microservices **register themselves** with Eureka.

Example:

```
user-service → register
order-service → register
product-service → register
```

Other services can **discover them dynamically**.

**Why it is needed**

Without Eureka:

```
service A must know exact IP of service B
```

With Eureka:

```
service A → ask Eureka → get service B location
```

This enables **dynamic scaling**.
